- [ ] **Prevent build from degrading command files**
  The /mill-build full rebuild regenerates command files from skill-commands.md spec, but the build agent paraphrases instead of preserving carefully-tuned output. Observed regressions: (1) numbered sequential steps replaced with flat bullets losing ordering, (2) combined commands (do-commit, finalize-do, finalize-do-commit, finalize-do-all) lost explicit script invocations and replaced them with vague 'Run do' references that the executor cannot resolve since it only sees its own command file, (3) dropped ## Rules sections and @taskmill skill references the executor needs, (4) list.md got hardcoded example numbers instead of placeholder N, (5) descriptions shortened losing clarity. Root cause: BUILD.md says 'implement according to the behavioral spec' for scripts but lacks equivalent guidance for commands — the build agent has no rule saying 'preserve existing command output structure'. Investigate: check git log for previous rebuilds (grep for 'mill-build' or 'rebuild' in commit messages) to see if earlier builds had the same drift problem. Solution options: (a) add a 'command generation rules' section to BUILD.md specifying numbered steps, explicit script paths, skill references, and ## Rules sections, (b) store golden command files as the source of truth instead of generating from spec, (c) add a diff-check post-build step that flags regressions
